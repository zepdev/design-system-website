## Basic Usage: Javascript

Import the javascript using your preferred method;

```js
//Import as ECMAScript Module
import ZEL from 'zeppelin-element-library'

// OR

//Require as CommonJs Module
const ZEL = require('zeppelin-element-library')
```

To get the JavaScript running, initialize ZEL after the DOM has loaded. Place the following script at the bottom of your body element or to your dedicated JavaScript file to initialize the library after every page load.

```html
<script>
  document.addEventListener('DOMContentLoaded', function(event) {
    ZEL.init()
  })
</script>
```

After initializing please make sure to use the refresh function after each DOM manipulation that effects any of the Zeppelin Element Library's elements to keep its functionality.

```js
ZEL.refresh()
```

If you use the Zeppelin Element Library inside a framework with Server Side Rendering or Static Site Generation like Gatsby you might see errors with "browser globals" like **window** or **document** (e.g. “window is not defined”).
If the code is in the render function of a React.js component, use React Hooks (React 16.8+) or move that code into componentDidMount which ensures the code doesn’t run unless it’s in the browser or simply check if window or document is defined.

```jsx
import React, { useEffect } from 'react';
import ZEL from 'zeppelin-element-library'

// Wrap the init in check for window
// init ZEL once
if (typeof window !== `undefined` && typeof document !== `undefined`) {
  document.addEventListener('DOMContentLoaded', function(event) {
    ZEL.init()
    window.ZEL = ZEL // if you want to access ZEL after uglification
  })
}

// React functional component
function Layout({something}) {

  // Effect Hook
  // Similar to componentDidMount and componentDidUpdate:
  useEffect(() => {
    // refresh ZEL on view change
    ZEL.refresh()
  })
```

## Basic Usage: Styles

To get started simply import the files you want to use in any project.

```js
import 'zeppelin-element-library/bundle/zeppelin-element-library.css'
```

All available css classes and javascript can be found under the Elements and Components sections of this website.
Simply use the css classes as normal.

```js
<button class="zep-button zep-button-primary">My Button</>
```

You can also use them along with other styling solutions. For example, in React;

```jsx
function MyButton({onClick}){
    return(
        <button className={classes.button,"zep-button"} onClick={onClick}>React Button</>
    )
}

export default MyButton
```

## Using theme.js

Also available from Zeppelin Element Library is a javascript object called theme. The theme file contains
everything outlined in the Guidelines section. These variables can easily be accessed and used in your project using any CCS-in-JS solution.

JSS in Gatsby example;

```jsx
// gatsby-config.js
const zeppelinTheme = require('zeppelin-element-library/bundle/themes/theme.json')
module.exports = {
  plugins: [
    {
      resolve: 'gatsby-plugin-jss',
      options: { zeppelinTheme },
    },
  ],
}

// MyComponent.jsx
import React from 'react'
import withStyles from 'react-jss'

const styles = theme => ({
  root: {
    marginBottom: theme.spacing.component.l.px * 2,
  },
  p: {
    color: theme.colors.gray.white.hex,
    background: theme.colors.corporate.indigoBlue.hex,
  },
})

const MyComponent = ({ classes }) => {
  return (
    <div className={classes.root}>
      <p className={classes.p}>This is a React Component!</p>
    </div>
  )
}

export default withStyles(styles)(MyComponent)
```
