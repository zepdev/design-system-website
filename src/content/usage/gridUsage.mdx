# Developers

## Implementation

The Zeppelin Design System grid is build with flexbox for modern browsers and has a fallback float grid for legacy browsers.
The grid is composed of a container: **.zep-grid**, rows: **.zep-grid\_\_row** and columns **.zep-grid\_\_col**.

A container has a 100% width and contains grid margins.
A row is the sub container to hold all individual elements, called columns. Columns MUST be used within a row.
Each column has horizontal padding (called a gutter) for controlling the space between them. This padding is then counteracted on the rows with negative margins.
The maximum amount of columns per row depends on the screensize.
Therefore it is necessary to define the column sizes based on this maximum amount.

<GridImage />

## breakpoint

There are six grid breakpoint to make the grid responsive, at: (xxs), xs, s, m, l, and xl.
Grid column breakpoint are based on minimum and maximum width media queries, and apply only to that breakpoint (not to those above or below). Above and below they will cover 100% of the row.

This example creates 3 columns using predefined grid classes, that will change width on differnt breakpoint.
The width of the columns is always a fixed percentage value, depending on the screensize and the related amount of colums.

## Column wrapping

Columns that exceed the provided space per row are wrapped onto a new line (but technically remain in the same row element).
All columns are always horizontally aligned left. Unused column space in the grid remains free.
The following example shows the result of a 3-row grid with different column sizes.
Feel free to change the screensize to see how the grid behaves.

## Alignment

Default alignment of columns is top left. This can be changed for browsers that support flex-box by using flexbox alignment utilities.
**The horizontal alignment should not be changed.**

### Vertical Alignment

Change the vertical alignment of columns with the addition of modifiers on **zep-grid\_\_row**.
